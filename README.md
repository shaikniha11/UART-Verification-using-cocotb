## UART-Verification-using-cocotb:
This project presents a complete RTL design and functional verification  of the UART (Universal Asynchronous Receiver Transmitter) protocol,  implemented in SystemVerilog and verified using the cocotb Python-based testbench framework with Icarus Verilog as the simulator and GTKWave as Waveform Generator.

UART is a serial communication protocol that transmits data asynchronously — without a shared clock between transmitter and receiver. Data is framed with a start bit, 8 data bits, and a stop bit. The receiver uses 16x oversampling to reliably detect and sample each bit at its center, making it robust against timing variations.

## The design is structured as five independent SystemVerilog modules integrated under a single top-level:
1.baud_rate_gen — Generates TX baud tick and RX 16x oversample tick from a 50MHz clock at 115200 baud rate
2.transmitter — 4-state FSM (IDLE → START → DATA → STOP) that serializes 8-bit data onto the TX line
3.receiver — 4-state FSM that deserializes incoming serial data using 16x oversampling to sample each bit at its center
4.sync_fifo — Synchronous FIFO with depth 8, full/empty flags, and simultaneous read/write support
5.uart_top — Top-level module integrating TX FIFO, transmitter, baud generator, receiver, and RX FIFO

## Verification Strategy
The verification environment is built using cocotb 2.0.1 with Icarus Verilog 12.0 as the simulator. A Python script automatically generates test vectors covering both edge cases and random values:
Edge cases: 0x00, 0xFF, 0xA5, 0x5A, 0x01, 0x80, 0xFE, 0x7F
Random cases: 10 randomly generated byte values per run
The loopback topology connects tx_out directly to rx, allowing end-to-end verification of the full TX-to-RX data path in a single simulation run.
